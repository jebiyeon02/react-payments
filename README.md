# 페이먼츠 미션

## 구현 사항 체크리스트
1. UI 및 컴포넌트 (Reusability)
- [ ] 반복되는 라벨, 설명 텍스트, 입력창, 에러 메시지 등을 공통 컴포넌트로 분리

- [ ] 컴포넌트별 전용 스타일과 글로벌 테마(Theme) 분리 적용

- [ ] 모든 입력 폼의 에러 발생 시 테두리 빨간색(#FF3D3D) 처리

- [ ] 모든 입력 폼의 포커스 시 테두리 검은색(#000000) 처리

2. 카드 프리뷰 (실시간 연동)
- [ ] 사용자가 입력하는 카드 번호 실시간 렌더링 (4자리 단위 묶음)

- [ ] 카드 번호 9~16번째 자리 마스킹 처리

- [ ] 만료일 실시간 렌더링 및 올바른 입력 시 / 기호 노출

- [ ] 카드 브랜드 조건 충족 시 실시간 로고(Visa / MasterCard) 표시

3. 기능 및 유효성 검사 (Readability & State)
- [ ] 카드 번호: 숫자만 입력 가능하도록 제한 및 실시간/필드 검증

- [ ] 만료일(월/년): 숫자만 입력 가능 및 유효한 범위(01~12 등) 검증

- [ ] CVC: 숫자 3자리 입력 제한 및 검증

- [ ] 상태 관리: 폼 라이브러리 없이 커스텀훅으로 폼 상태 중앙 관리

- [ ] 에러 처리: 에러 로직을 뷰 컴포넌트와 분리하여 명확한 가독성 확보

4. 테스트 및 기타 제약사항 (Test & Library)
- [ ] Storybook을 활용하여 주요 컴포넌트(CardPreview, Input 등)의 다양한 상태 시각적 테스트

- [ ] 외부 폼 라이브러리 미사용

- [ ] styled-components를 활용한 동적 스타일링 적용

## 기능 요구 사항
| 기능 분류 | 상세 요구 사항 |
| :--- | :--- |
| **카드 번호 입력 및 식별** | • 사용자가 입력하는 카드 번호를 실시간으로 파악하여 브랜드 로고 UI에 표시합니다.<br>• 입력은 숫자만 가능하며, 유효하지 않은 번호 입력 시 피드백(에러 텍스트 및 빨간 테두리)을 제공합니다. |
| **카드 브랜드 구분 로직** | • **Visa:** `4`로 시작하는 16자리 숫자<br>• **MasterCard:** `51`~`55`로 시작하는 16자리 숫자 |
| **카드 유효기간 입력** | • 월과 년도를 범위 내에서만 입력할 수 있어야 하며, 숫자만 입력하도록 제한합니다. |
| **실시간 프리뷰 업데이트** | • 사용자의 카드 정보 입력에 따라 카드 프리뷰 화면이 실시간으로 동기화됩니다. |



## 폴더 구조 설계 (Folder Structure)
도메인 응집도와 유지보수성을 높이기 위해 비즈니스 로직과 UI를 철저히 분리했습니다.

```
src/
 ├── common/                 // 전역 공통 컴포넌트 및 스타일
 │    ├── components/        // Description, Input, Label, Title
 │    └── styles/            // Theme, GlobalStyles
 ├── feature/                // 도메인 기능 영역
 │    └── CardRegister/      // 카드 등록 도메인
 │         ├── components/
 │         │    ├── CardPreviewSection/  // 시각화 영역 (Dumb)
 │         │    └── InfoInputSection/    // 폼 입력 영역 (Dumb)
 │         ├── hooks/        // 폼 상태 및 에러 중앙 통제 (Smart)
 │         ├── utils/        // 순수 유효성 검사 및 포맷팅 로직
 │         ├── types/        // 인터페이스 정의
 │         └── index.tsx     // 페이지 진입점
 └── App.tsx
```
